## Требования к выполнению и оценке

Ниже представлены требования которым должно удовлетворять задание по разработке приложений с использованием семафоров для получения соответствующей оценки.

При отсутствии корректной реализации, использующей семафоры задача считается нерешенной (0 баллов).

Для выполнения задания на более высокую оценку необходимо разработать и представить в отдельном каталоге программу **для каждой нижестоящей оценки**. Допускается единый набор тестовых файлов (если они необходимы для понимания программы). Но результаты тестирования (даже если они одинаковы) размещать в подкаталоге той программы, для которой проводится тестирование.

Так как большинство задач связаны с моделированием различных ситуаций, необходимо обеспечить наглядное представление результатов. Для интерактивного отображения на дисплее рекомендуется использовать временные задержки. Рекомендуется использовать аргументы командной строки для запуска заданного числа процессов. Также для параллельного запуска множества процессов допускается использование соответствующих скриптов, в котором программы запускаются в фоновом режиме. В целом все определяется условием задачи и фантазией разработчика.

**Программы должны быть написаны на языке программирования C или C++ с использованием семафоров, входящих в стандарты POSIX. Обязательным условием является выполнение программ в ОС Linux.**

## Общие требования

Общие требования связаны с оформлением отчета и реализаций параметров, общих для всех задач. К ним относятся:

1. В отчете необходимо привести фамилию, имя, отчество исполнителя, группу.
2. Привести номер варианта и условие задачи.
3. Представить сценарий решаемой задачи поясняющий, каким образом исходные сущности и их поведение отображаются в процессы и их взаимодействие.
4. Необходимо реализовать завершение программы в соответствии с условием задачи, а также предусмотреть корректное завершение программы по прерыванию с клавиатуры по соответствующему сигналу.
5. В программе предусмотреть удаление семафоров и разделяемой памяти по ее завершению любым из способов.
6. Для каждой из разработанных программ в отчете представить способ ее запуска. Особенно в тех ситуациях, когда порядок запуска отдельных процессов имеет значение.
7. Когда число процессов реализующих то или иное действие не является точно заданным, необходимое значение можно задавать в качестве аргумента командной строки программы (при запуске дочерних процессов) или путем ручного запуска произвольного числа независимых процессов.
8. Результаты работы программы должны быть отражены в отчете.

## Первая программа (4–6 баллов)

**Первая программа.** Разработать консольное приложение, в котором единый родительский процесс запускает и использует требуемое число только дочерних процессов.

Требования:

1. Представить сценарий решаемой задачи поясняющий, каким образом исходные сущности и их поведение отображаются в процессы и их взаимодействие.
2. Множество процессов взаимодействуют с использованием **неименованных** POSIX семафоров.
3. Обмен данными ведется через разделяемую память в стандарте POSIX.
4. Реализовать завершение программы в соответствии с условием задачи, а также предусмотреть корректное завершение всей программы по прерыванию с клавиатуры при подаче соответствующего сигнала.
5. Каждый из процессов выводит в общий поток данных информацию необходимую для понимания его работы в соответствии с выполняемой им ролевой функцией.
6. В программе предусмотреть удаление семафоров и разделяемой памяти по ее завершению любым из способов.
7. Результаты работы программы должны быть отражены в отчете.

## Вторая программа (7–8 баллов)

**Вторая программа.** В дополнение к ранее написанной программе разработать еще одно приложение, состоящее из отдельных программ–процессов, запускаемых независимо друг от друга в разных консольях.

Требования:

1. Множество процессов взаимодействует с использованием **именованных POSIX** семафоров.
2. Обмен данными ведется через разделяемую память в стандарте POSIX.
3. Для завершения приложения использовать те же подходы, что и в предыдущих решениях.
4. Каждый из процессов выводит в свою консоль информацию необходимую для понимания его работы в соответствии с выполняемой им ролевой функцией.
5. В отчет расширить информацией о добавленной реализации и привести соответствующие результаты работы программы.

---

## Третья программа (9 баллов)

**Третья программа.** К предыдущей программе добавить дополнительный процесс–наблюдатель, который интегрирует и корректно отображает всю информацию, ранее разработанных независимых процессов.

Требования:

1. Для завершения приложения использовать те же подходы, что и в предыдущих решениях.
2. Все процессы запускаются независимо друг от друга. Процесс–наблюдатель получает информацию через именованный канал или очередь сообщений. Ранее разработанные процессы, дополнительно с отображением в своей консоли, передают порождаемую ими информацию наблюдателю.
3. Отчет расширить информацией о добавленной реализации и привести соответствующие результаты работы программы.

---

## Четвертая программа (10 баллов)

**Четвертая программа.** В дополнение к программам на предыдущие оценки необходимо разработать программу, допускающую подключение не одного, а нескольких наблюдателей, каждый из которых выводит информацию в свою консоль.

Требования:

1. Для завершения приложения использовать те же подходы, что и в предыдущих решениях.
